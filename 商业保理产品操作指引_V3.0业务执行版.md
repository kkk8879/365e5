# 商业保理产品操作指引

版本：V3.0（业务执行版）

编制部门：产品设计与风险管理部

适用范围：全国【陕西—大湾区为核心拓展区】

---

## 第一章 总则

### 一、编制目的

本指引系公司商业保理业务的操作规范文件，旨在明确各产品线的准入条件、操作流程、风险控制要求及禁入标准，为业务人员展业提供可执行的判断依据和操作规范，确保业务开展在风险可控的前提下有序推进。

### 二、风控总纲

**资产真实性优先，全流程资金闭环。**

商业保理业务的第一还款来源为债务人对应收账款的支付，而非债权人或担保人的一般偿债能力。风控逻辑应坚持"先核资产、再谈保理；先锁回款、再谈授信；先排冲突、再谈入池"的原则，将应收账款真实性、合法性、可转让性、权利完整性、价值确定性和可回收性审查置于主体信用评价之前。

全流程资金闭环是底线要求，但资金闭环的前提是底层资产真实、合法、可执行。脱离资产真实性谈资金闭环，本质上是将保理异化为信贷。

### 三、产品体系

| 序号 | 产品模式 | 场景属性 | 风控核心 |
|------|----------|----------|----------|
| 1 | 商品包销保理 | 销售场景 | 控货+控销售回款+应收验真 |
| 2 | 电商应收保理 | 销售场景 | 数据直连+退货率管控+回款账户锁定 |
| 3 | 基建保理 | 采购场景 | 确权/确真+财政预算+回款路径锁定 |
| 4 | 产业通用保理（明/暗保理） | 采购场景 | 主体授信+应收账款六性审查 |
| 5 | 订单融资 | 采购/销售双场景 | 全流程嵌入管控+主体信用兜底 |

## 第二章 通用风控要求

### 一、应收账款审查"六性"标准

所有产品线均须对应收账款进行以下审查，审查结论须形成书面记录并归入项目档案：

| 审查维度 | 审查内容 | 审查方式 |
|----------|----------|----------|
| 真实性 | 基础交易是否真实发生，应收账款是否实际形成 | 交叉验证：合同+发货/物流+验收+发票+纳税申报+银行流水，至少三组以上交叉匹配 |
| 合法性 | 基础合同是否合法有效，应收账款形成是否具备法律依据 | 审查合同效力、交易背景、行业资质；关注关联交易公允性 |
| 有效性 | 债权人是否已完全履行合同主要义务，应收账款是否已到期或确定到期 | 审查履约凭证（验收单、监理报告、签收单等）；未来应收须满足"有基础法律关系"条件 |
| 可转让性 | 应收账款是否可依法转让，是否存在禁止转让条款 | 审查基础合同转让条款；民法典第545条适用分析；完成中登网转让登记 |
| 权利完整性 | 应收账款是否存在质押、查封、冻结、重复融资等权利负担 | 中登网查询、征信查询、实地核实、出质人书面确认；已质押资产不得入池 |
| 价值确定性 | 应收账款金额是否确定、账期是否明确、回款可能性是否可评估 | 审查合同金额与发票一致性、历史结算记录、付款习惯、回款预测模型 |

### 二、通用禁入清单（一票否决项）

符合以下任一条件的业务，不得办理：

1. **应收账款不真实或无法验证**：基础交易无法通过交叉验证确认真实性
2. **基础合同存在禁止转让条款**且无法通过法律途径解决（暗保理须另行评估通知时机）
3. **应收账款已质押、查封或存在权利冲突**且无法解除或取得质权人书面同意
4. **回款路径无法锁定**：债务人不同意变更回款账户或无法签订三方监管协议
5. **关联交易缺乏商业合理性**：买卖双方存在关联关系且交易对价明显不公允
6. **债务人已被列为失信被执行人或存在重大涉诉风险**
7. **基础合同未经有权机关备案（涉财政资金项目）**
8. **以票据或其他有价证券产生的单纯付款请求权**
9. **债权人主要义务仅为给付金钱（如借贷、保证金）产生的应收账款**
10. **应收账款权属不清或存在争议**

### 三、通用放款前提条件

1. 应收账款转让登记已完成（中登网）
2. 债务人确认/转让通知已送达（明保理须取得书面确认，暗保理须留存通知送达凭证）
3. 回款监管账户已开立且保理方已取得账户控制权（U盾、密码、印鉴）
4. 担保/增信措施已签署生效
5. 业务审批流程已完成且审批条件已落实
6. 法律审查意见已出具且无重大法律障碍

### 四、通用保后管理要求

| 管理事项 | 频次/要求 |
|----------|-----------|
| 回款到账监控 | 逐笔核对，T+1确认 |
| 间接付款识别与处理 | 发现间接付款立即预警，要求债务人限期纠正，逾期未纠正启动违约条款 |
| 应收账款到期催收 | 到期前5个工作日发送付款提醒函 |
| 保后现场检查 | 首次放款后30日内完成首次检查；常规每季度不少于1次 |
| 逾期管理 | 逾期1日电话催收；逾期7日书面催收；逾期30日律师函；逾期90日认定为不良资产 |
| 风险预警 | 债务人/债权人涉诉、被执行、经营异常、重大舆情等即时预警 |
| 中登网续登记 | 到期前30日完成续登记 |

## 第三章 场景一：商品包销保理

### 一、产品定义与适用范围

商品包销保理系指包销方将其基于包销协议对下游销售形成的应收账款转让给保理公司，由保理公司提供融资服务的业务模式。核心风控逻辑为：**控货+控销售回款+应收验真**。

### 二、场景准入条件

| 序号 | 准入条件 | 性质 |
|------|----------|------|
| 1 | 包销方须提供已签署的包销协议或代理协议，协议项下权利义务清晰、可转让 | 硬性条件 |
| 2 | 包销商品须具备可识别、可计量、可管控的物理特征，能够实现货权控制 | 硬性条件 |
| 3 | 包销方须提供下游销售渠道证明（合同、订单、历史结算记录），且下游回款路径可锁定 | 硬性条件 |
| 4 | 融资期限与商品去化周期匹配，原则上不超过90天（矿产原料等特殊品类另行审批） | 硬性条件 |
| 5 | 单笔融资金额原则上不超过包销方净资产的50% | 硬性条件 |
| 6 | 包销方历史经营数据可查证（至少近6个月银行流水或销售台账） | 硬性条件 |
| 7 | 建议包销方具备相关行业经营经验1年以上 | 风控建议 |

### 三、各品类操作要点

#### （一）景区门票

| 项目 | 要求 |
|------|------|
| 确权/确真方式 | 包销协议+OTA平台核销流水（近3个月）+景区门票系统后台数据 |
| 回款控制 | 绑定OTA平台结算账户为保理监管账户；二道包销须查代理商核销流水 |
| 质押率 | 预付+分成模式下，预付比例≤50% |
| 风控红线 | 景区年客流量可查证；包销协议剩余期限≥融资期限+3个月 |

#### （二）酒店房源

| 项目 | 要求 |
|------|------|
| 确权/确真方式 | 包销协议+会展排期表+往届订房数据；或TMC企业差旅合同 |
| 回款控制 | OTA平台结算账户/会展公司对公账户绑定监管 |
| 质押率 | 预付比例≤50%，酒店须自担50%销售风险 |
| 风控红线 | 酒店须提供近两年场地使用率数据；包销方与酒店须为非关联方（关联方须专项审查商业合理性） |

#### （三）停车收费

| 项目 | 要求 |
|------|------|
| 确权/确真方式 | 停车场经营权/租赁合同+收费系统后台数据+历史6个月银行流水 |
| 回款控制 | 收费系统后台只读/监管权限+收款码/商户号归集至监管账户；签订收费权质押合同；代运营方不接触资金 |
| 质押率 | 可核验净现金流折算，折扣率≤50% |
| 风控红线 | 逐项审查原始合同是否允许收费权/收益权转让；排查既有质押冲突（中登网+实地核实）；原则上仅纳入剩余合同期限充足、权利清晰、历史经营稳定的项目 |

#### （四）婚庆场地

| 项目 | 要求 |
|------|------|
| 确权/确真方式 | 场地包销协议+酒店近两年场地使用率数据+已签婚庆订单汇总 |
| 回款控制 | 酒店回款账户监管+分销婚庆公司回款归集 |
| 质押率 | 预付比例≤50%，酒店须自担50%销售风险 |
| 风控红线 | 融资方为酒店/婚礼堂（非轻资产婚庆公司）；酒店须承诺场地分销 |

#### （五）酒类

| 项目 | 要求 |
|------|------|
| 确权/确真方式 | 包销协议+下游烟酒店/餐饮店订单（≥50家）+物流发货单（货已到终端） |
| 回款控制 | 绑定终端渠道回款账户；按订单量逐笔放款 |
| 单笔限额 | 单笔≤100万元 |
| 融资期限 | 节庆短周期包销，≤60天 |
| 风控红线 | 仅限光瓶口粮酒（30-80元/瓶）或低度快消酒饮；按物流发货单放款，不按库存放款；白酒市场下行期审慎介入 |

#### （六）特色农产品

| 项目 | 要求 |
|------|------|
| 确权/确真方式 | 包销协议+冷库仓单+冷库监管协议（冷库方凭保理公司指令放货） |
| 回款控制 | 仓单质押+动态赎货（卖一批赎一批）；下游回款账户监管 |
| 质押率 | ≤50%（水果价格波动大） |
| 风控红线 | 必须指定第三方冷库并签署三方监管协议；冷库方凭保理公司书面指令方可放货，不得接受包销方单方指令 |

#### （七）生鲜

| 项目 | 要求 |
|------|------|
| 确权/确真方式 | 产地端分级包销协议+冷链物流单+商超验收单+下游采购商预付凭证（≥30%） |
| 回款控制 | 下游采购商回款至监管账户；约定下游拒收时包销方仍须按协议还款 |
| 融资期限 | ≤30天 |
| 风控红线 | 不做下游赊销的包销方；只做产地端分级包销，直接发往商超渠道（盒马等） |

#### （八）工业材件

| 项目 | 要求 |
|------|------|
| 确权/确真方式 | 集采协议+下游采购订单汇总表+仓库货权凭证；价格挂钩期货价格 |
| 回款控制 | 控仓库+按订单量逐笔放款 |
| 融资期限 | 单品单批，周期≤90天 |
| 风控红线 | 集采平台须提供历史采购数据；放款与下游订单一一对应，禁止资金挪用 |

#### （九）光伏组件

| 项目 | 要求 |
|------|------|
| 确权/确真方式 | 分布式工商业屋顶EPC包销协议+并网文件+业主已付部分工程款凭证 |
| 回款控制 | 电费收入/业主付款归集至监管账户 |
| 单笔限额 | 单个项目50-200万元 |
| 风控红线 | 仅做分布式工商业屋顶小EPC；须已并网且补贴文件到手；EPC包销方为安装商的，须提供业主已付部分工程款凭证 |

#### （十）电子产品

| 项目 | 要求 |
|------|------|
| 确权/确真方式 | 长期供货协议+下游采购订单+物流签收单 |
| 回款控制 | 下游回款账户监管 |
| 风控红线 | 仅做单一客户长期合作；客户注册资本过低的，须追加实控人房产抵押或法人连带担保+保险 |

#### （十一）矿产原料

| 项目 | 要求 |
|------|------|
| 确权/确真方式 | 下游长协合同（如陕煤集团等大型国企）+矿企发货通知书+货权凭证 |
| 回款控制 | 货物存入保理公司指定仓库（可租临时场地），不得留在矿场；下游回款至监管账户 |
| 融资期限 | 周期＞1年的须专项审批 |
| 风控红线 | 仅做大型国企下游锁定的长协业务；须取得下游长协合同+矿企发货通知书；无价格对冲机制的审慎介入 |

#### （十二）医疗器械

| 项目 | 要求 |
|------|------|
| 确权/确真方式 | 供货协议+三甲医院应收账款确权函（医院财务科盖章，不接受科室章）+耗材已进入医院SPD系统使用记录 |
| 回款控制 | 医保回款账户/医院付款账户监管 |
| 融资期限 | 回款方为三甲医院且医保回款周期＜12个月 |
| 风控红线 | 一般情况不做；确权函必须由医院财务科盖章；耗材须进入SPD系统可查使用记录；医保回款周期＞12个月的不做 |

### 四、场景禁入条件（商品包销通用）

1. 包销方无法提供可验证的下游销售渠道或历史销售数据
2. 商品不可控（无法实现货权控制或仓储监管）
3. 去化周期与融资期限严重不匹配
4. 包销方与供货方存在关联关系且交易定价明显不公允
5. 商品属于限制流通物或需要特许经营资质但包销方不具备

## 第四章 场景二：电商应收保理（"店融通"）

### 一、产品定义

电商应收保理系基于电商平台商家在平台已形成且待结算的应收账款，由保理公司提供短期融资服务的业务模式。核心风控逻辑为：**数据直连+退货率管控+回款账户锁定**。

### 二、准入条件

| 序号 | 准入条件 | 性质 |
|------|----------|------|
| 1 | 商家经营平台：抖音/淘宝/京东/拼多多/快手 | 硬性条件 |
| 2 | 单店月销售额≥100万元 | 硬性条件 |
| 3 | 店铺经营时间≥6个月 | 硬性条件 |
| 4 | 近3个月无平台违规记录 | 硬性条件 |
| 5 | 退货率≤15% | 硬性条件 |
| 6 | 店铺后台数据可对接（订单/物流/结算） | 硬性条件 |

### 三、产品要素

| 项目 | 内容 |
|------|------|
| 融资比例 | 应收账款金额的≤70% |
| 融资期限 | ≤30天，随借随还 |
| 放款时效 | T+1 |
| 费率 | 按次收费，具体以审批为准 |

### 四、风控要点

1. **数据直连**：对接店铺后台，实时获取订单、物流、结算数据，与商家自主申报数据交叉验证
2. **回款账户锁定**：绑定平台唯一结算账户为保理监管账户，保理款自动扣划，禁止变更回款路径
3. **退货率监控**：退货率＞15%时暂停新增放款；退货率＞20%时启动存量清收
4. **平台处罚监控**：近3个月存在虚假交易、刷单等违规处罚的，不得准入
5. **多平台交叉验证**：同一商家多平台经营的，合并计算销售额及退货率

### 五、保后管理特殊要求

| 管理事项 | 频次/要求 |
|----------|-----------|
| 平台后台数据巡检 | 每日 |
| 退货率变动监控 | 每周 |
| 回款到账核对 | 逐笔T+1 |
| 平台违规/处罚监控 | 实时 |
| 店铺经营异常预警 | 即时（停业、转让、降级等） |

## 第五章 场景三：基建保理（"六张网"）

### 一、产品定义

基建保理系基于基础设施建设项目中供应商对总包方/业主方已形成的应收账款，由保理公司提供融资服务的业务模式。核心风控逻辑为：**确权/确真+财政预算+回款路径锁定**。

### 二、"四不接"铁律（一票否决）

| 序号 | 铁律内容 |
|------|----------|
| 1 | ❌ 合同未在政府平台备案的不接 |
| 2 | ❌ 总包方不配合提供盖章版验收单（纸质或PDF）的不接 |
| 3 | ❌ 财政预算未明确的不接 |
| 4 | ❌ 应收账款未经交叉验证的不接（不得仅凭一纸验收单即确认应收） |

### 三、通用准入条件

| 序号 | 准入条件 | 性质 |
|------|----------|------|
| 1 | 项目已纳入省级以上重点项目计划或财政预算安排 | 硬性条件 |

| 6 | 中登网查询应收账款无既有质押或权利冲突 | 硬性条件 |

### 五、基建保理专项风控要求

1. **验收单核验**：须取得总包方盖章版验收单（纸质或PDF），不接受电子邮件扫描件+现场拍照替代。验收单内容须与合同约定交付内容一致
2. **财政预算查询**：须核实项目是否纳入当年财政预算，预算金额是否覆盖应收账款金额。涉区县财政的，须关注地方债务风险等级
3. **间接付款监控**：债务人向非监管账户付款的，视为间接付款，即时预警并要求纠正；连续发生间接付款的，启动违约条款
4. **进度跟踪**：关注项目实际施工进度与合同约定进度的匹配情况，进度严重滞后须专项评估

## 第六章 场景四：产业通用保理（明保理/暗保理）

### 一、产品定义

产业通用保理系基于供应商与核心企业之间因货物/服务交易已形成的应收账款，由保理公司提供融资服务的业务模式。核心风控逻辑为：**应收账款六性审查+主体信用评价**，其中应收账款审查权重不低于主体信用评价。

### 二、明保理与暗保理的适用条件

| 项目 | 明保理 | 暗保理 |
|------|--------|--------|
| 适用条件 | 债务人配合确权且同意变更回款路径 | 债务人不便确权或通知，但应收账款真实可验证 |
| 确权要求 | 取得债务人书面确认（盖章版应收账款确认函） | 不通知债务人，但须留存转让通知送达凭证备查 |
| 回款控制 | 债务人直接付款至保理监管账户 | 债权人收到款项后划转至保理账户，须监控债权人结算账户 |
| 转让登记 | 中登网登记 | 中登网登记 |
| 风险差异 | 回款路径确定，风险较低 | 存在间接付款风险，回款可控性偏弱 |

### 三、禁转条款处理

基础合同存在禁止转让条款的，按以下原则处理：

1. **明保理**：须与债务人协商修改合同条款，删除禁止转让约定或取得债务人书面同意转让函。无法取得的，不得办理
2. **暗保理**：依据民法典第545条，金钱债权不得转让的约定不得对抗第三人。但须注意：（1）该条款适用于金钱债权，非金钱债权不适用；（2）虽可对抗债务人，但不影响债权人与债务人之间的违约责任，须评估债权人因违约可能产生的损失对还款能力的影响；（3）建议在保理合同中约定债权人承担因违反禁转条款产生的全部损失

### 四、"三单合一"核验标准

暗保理项下应收账款审查须满足"三单合一"要求，且至少两组以上交叉验证：

| 核验单据 | 核验要点 |
|----------|----------|
| 合同 | 合同金额、标的、交付标准、付款条件、账期与应收账款金额、期限一致 |
| 发票 | 发票查验平台验真；发票金额、品名、数量与合同一致；开票时间在合同约定期限内 |
| 发货/签收/验收单 | 物流单据或签收凭证或验收报告；内容与合同约定交付物一致；签收/验收时间在发票开具之前或同期 |

### 五、行业准入参考

| 行业 | 目标客户 | 确权方式 | 风控要点 |
|------|----------|----------|----------|
| 装备制造 | 汽配/机械零部件供应商 | 核心企业应付账款确权（明保理）或三单合一（暗保理） | 发票查验、账期匹配、历史交易流水 |
| 医药流通 | 药品/耗材供应商 | 医院确认函（财务科盖章） | 医院回款周期、SPD系统记录、退货率 |
| 建材 | 水泥/钢材/管材供应商 | 总包方进场验收单+合同 | 项目备案、财政预算、验收单盖章核验 |
| 能源 | 电力/光伏设备供应商 | 业主方验收确认 | 并网文件、电费收入覆盖 |

## 第七章 场景五：订单融资

### 一、产品定义

订单融资系基于供应商已取得买方采购订单但尚未形成应收账款的阶段，由保理公司提供融资服务的业务模式。本质：**没有应收账款，只有订单**，须在采购/生产/交付全流程嵌入管控，同时要求主体信用兜底。

### 二、准入条件

| 序号 | 准入条件 | 性质 |
|------|----------|------|
| 1 | 买方/卖方至少一方信用评级A+以上，或提供足额资产抵押 | 硬性条件 |
| 2 | 订单真实可验证（买方书面确认或招投标文件可查） | 硬性条件 |
| 3 | 实际控制人连带担保 | 硬性条件 |
| 4 | 回款账户共管或监管 | 硬性条件 |
| 5 | 订单不得转包 | 硬性条件 |
| 6 | 供应商与买方存在≥1年的历史合作记录 | 硬性条件 |

### 三、产品要素

| 项目 | 采购订单（垫资采购） | 销售订单（备货/生产） |
|------|----------------------|----------------------|
| 融资比例 | 50%-60% | 50%-60% |
| 过程管控 | 监控原材料采购凭证、到货视频、受托支付至上游 | 监控生产进度周报、质检报告、物流轨迹、仓储巡查 |
| 放款节奏 | 按采购节点分批放款 | 按生产/交付节点分批放款 |

### 四、规模管控

| 项目 | 要求 |
|------|------|
| 首年投放上限 | 不超过公司总投放的20% |
| 单笔上限 | ≤200万元 |
| 优先选择 | 与核心企业合作稳定的老客户 |

### 五、退出机制

| 触发条件 | 处理方式 |
|----------|----------|
| 订单融资不良率＞3% | 暂停新增投放，排查存量 |
| 单笔逾期＞30天 | 启动催收，不再对该客户新增订单融资 |
| 底层订单连续2个月未按期交付 | 要求提供书面说明及补救方案；无法补救的启动回购 |
| 买方/卖方主体信用重大恶化 | 暂停新增，评估存量风险 |

### 六、风险警示

订单融资不良率通常高于其他四类产品。核心原因在于融资时点早于应收账款形成，资产确定性最弱。业务开展须坚持：

1. **先试后扩**：首次合作客户先做1-2单小额试跑，观察交付能力和回款纪律后再逐步扩量
2. **全程管控**：融资期间须持续跟踪订单执行进度，不得放款后疏于管理
3. **主体兜底**：必须同时评估买方和卖方的主体信用，至少一方具备较强的偿债能力

## 第八章 合规审查要求

### 一、合规审查要点

1. **业务定性**：每个项目须明确属于标准保理业务还是其他结构化融资。存在"名为保理、实为借贷或收益权融资"风险的，须由法务和合规部门专项会审
2. **商品包销场景**：控货+销售网的结构须确保基础交易具备真实商业目的，避免被认定为存货质押融资套用保理之名
3. **基建保理场景**：须核实项目是否具备合规审批手续，避免介入违规建设项目
4. **订单融资场景**：须确保融资依据为真实订单而非框架协议或意向性订单
5. **未来应收账款**：仅接受"有基础法律关系"的未来应收账款（如已签署合同、已确定收费标准），不接受"无基础法律关系"的未来应收账款（如未来可能的销售收入的融资）

### 二、法律审查专项要求

| 审查事项 | 要求 |
|----------|------|
| 合同效力审查 | 基础合同是否合法有效，是否存在无效或可撤销情形 |
| 转让条款审查 | 是否存在禁止转让条款，如有须按第六章第二节处理 |
| 权利冲突排查 | 中登网查询+征信查询+出质人书面确认 |
| 通知与登记 | 明保理须完成转让通知并取得确认；暗保理须留存通知凭证；中登网登记须在放款前完成 |
| 担保效力审查 | 担保决议、授权文件是否完备；上市公司须关注公告义务 |

## 第九章 审批权限与流程

### 一、审批权限

| 项目类型 | 审批层级 |
|----------|----------|
| 单笔融资金额≤500万元 | 风控总监审批 |
| 500万元＜单笔融资金额≤2000万元 | 总经理审批 |
| 单笔融资金额＞2000万元 | 董事会审批 |
| 订单融资项目 | 不论金额，均须总经理审批 |
| 新场景/新品种首笔业务 | 不论金额，均须总经理审批 |

### 二、审批流程

```
业务部门立项 → 风控部门审查（含法律审查） → 审批层级审批 → 放款前提条件落实 → 放款 → 保后管理
```

风控审查须出具书面审查意见，明确列示：准入条件满足情况、六性审查结论、主要风险点及缓释措施、是否建议审批及审批条件。
